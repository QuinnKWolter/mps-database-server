Search

Searching the Database

  • From the home page, you can perform a Global Search with the provided text input
  • To search, type a query in the search input field and hit enter or click the blue Search button
  • As you type in the search bar, suggestions will appear
    Clicking a suggestion will place a term broader than the suggestion into the search bar
    This is done so that clicking a search suggestion for, say, a drug trial, will also display other results with the compound in question
  • Clicking the white button reading Global let's you switch to a Bioactivites Search described here
  • After submitting a global query, you are brought to the Search Results page
    This page has a text input so that you can submit another global search
    Below the text input, you will see a number of checkboxes with labels for different types of data
    Clicking a checkbox will restrict the results to only display results that have a checked-off type
    You can clear all of the checkboxes and show all of the results again by clicking the red Clear Filters button

Studies

Studies

Adding or Editing a Study

  • Studies serve to organize plates, chips, and their related data
  • The Study Type indicates the general purpose of the study
    Multiple Study Types can be chosen and at least one is required
  • Please try to name the study informatively
    e.g. "Kidney-Liver Vitamin C Metabolism" is more useful than "Study-1"
  • You may not edit the Study ID, for it is automatically generated from your other choices
  • Choosing to Restrict Access will hide the study from anyone who isn't in the listed group
  • Checking Use in Calculations tells the database to use this study's data in predictive modeling and other calculations
  • You can upload an image, such as a picture of the setup or an illustration of the protocol
    If a preview of the uploaded image does not appear, something went wrong
  • You can upload a PDF or word file which contains the study's Protocol
  • An Integrated Study Configuration needs to be specified for integrated studies
    If the Integrated Study Configuration you need is not available, click the provided link to add one

Adding or Editing Assays

  • The section entitled Assays gives a list of assays performed for the study
  • The column for Target/Analyte specifies what the assay was analyzing, e.g. the protein Albumin
    If a target you need is not on the list, please contact an administrator or post in Feedback
  • The column for Method/Kit shows how the assay was performed
    If a method you need is not on the list, please contact an administrator or post in Feedback
  • The column for Readout Unit shows what units the data (namely the data you wish to upload) from the assay were recorded in
    If a unit you need is not on the list, please contact an administrator or post in Feedback
  • If you need another row, click the Add Assay button

Adding Supporting Data

  • You can add Supporting Data such as the raw values used to acquire the data uploaded in the Readouts
  • This data will not be used in calculations and does not need to be formatted in a specific way
  • If you need another row, click the green "Add Supporting Data" button
  • If you still need to acquire this data, you can edit the study later and add it then

Integrated Study Configurations

Adding or Editing an Integrated Study Configuration

  • An Integrated Study Configuration shows how different devices in an integrated study associate with eachother
  • Please be sure to use an informative name

Defining How the Models are Associated

  • Each model has a Label which can be one or two characters long
  • Organ specifies which organ model is associated with the label
  • Sequence Number shows the order of input for the models
  • Output lists all of the other models the current model feeds into
    Output should be a list of labels separated by dashes: e.g. "AB-CD" means the model is connected to AB and CD
  • Coupling indicates whether or not the model is directly hooked up to the other models
    Physical shows that the models are physically connected while Functional shows that the model is not receiving media directly
  • Click Add Model to add another row

Chip Setups

Adding or Editing a Chip Setup

  • A Chip Setup describes the protocols taken to prepare a chip
  • After selecting an Organ Model, a link to view its standard protocol will appear (granted a PDF has been uploaded)
  • Variance from Protocol should list the ways this setup diverged from the organ model's standard protocols
  • The Chip ID/Barcode can not be the same as another chip in the study
  • Test Type specifies whether the chip functions as a control or not {#
    #} {# If the chip is not a control chip, you must specify a compound, concentration, and unit#}

Adding Compounds

  • Each row under the header Compounds describes a compound added to the chip
  • If this chip does not have any compounds, do not modify this section
  • To get another row, click the Add Compound button
  • Be sure to specify the supplier, lot, receipt date, concentration (and unit), addition time, and duration
  • If necessary, please use "N/A" for suppliers and lots that are unknown
  • If necessary, please enter nothing into the receipt date if the date is unknown
  • Suggestions for supplier and lot will be shown depending on your previous selections
  • Suggestion lists are shortened depending on what you type into the text boxes
  • Please choose applicable suggestions when possible
  • Use 'e' for scientific notation: e.g. 1e+3 = 1000

Adding Cells

  • Each row under the header Cell Samples describes cells added to the chip
  • If this chip does not have any cells, do not modify this section
  • To get another row, click the Add Cells button
  • Click a magnifying glass to see a list of available cell samples then click the desired row to select that sample
  • Use 'e' for scientific notation: e.g. 1e+3 = 1000
  • If you need a cell sample that is not listed, click the provided link to add it

Flagging for Review

  • Clicking flags the current setup for review
  • Clicking Notes brings up an input dialog where you can explain why this setup was flagged
  • Clicking removes the flag from the current setup

Chip Readouts

Adding or Editing a Chip Readout

  • A Chip Readout describes the assays performed and the data from these assays

Adding or Editing Assays

  • Before uploading data, please be sure to add every assay performed in the section entitled Assays
    You can change this list by going to the Study's edit page and finding the section Assays
    Please see Study

Uploading Data

  • If you are uploading data for a single Readout, use the .csv or .xlsx format

    In Excel, you will see an option reading "CSV (Comma delimited)" in the "save as type" drop down in the "save as" menu
    If you want to upload data for multiple Readouts, see Uploading an Excel File of Readout Data

  • The format for chip uploads is as follows:

    Chip ID Cross Reference Assay Plate ID Assay Well ID Day Hour Minute Target/Analyte Method/Kit Sample Location Value Value Unit Replicate Caution Flag Exclude Notes
    [Chip ID] [Cross Reference] [Assay Plate ID] [Assay Well ID] [Day] [Hour] [Minute] [Target/Analyte] [Method/Kit] [Sample Location] [Value] [Value Unit] [Replicate] [Caution Flag] [Leave Blank to Pass] [Notes]
    [Chip ID] [Cross Reference] [Assay Plate ID] [Assay Well ID] [Day] [Hour] [Minute] [Target/Analyte] [Method/Kit] [Sample Location] [Value] [Value Unit] [Replicate] [Caution Flag] [Any Value in this Column Marks the Reading Invalid] [Notes]
  • File uploads can use either full names ("Albumin") or abbreviations ("ALB") for Target/Analytes
  • Some assays involve transferring media from a device into a plate
    For these assays, specify this information with Assay Plate ID and Assay Well ID
  • Please note that time is split into Day, Hour, and Minute
  • The Sample Location indicates from where in a device the tested sample was acquired
    For instance, if an Organ Model is split into a top and bottom compartment, one might put terms like "Apical" or "Basal" in the Sample Location column
  • Rows in red will be excluded
    Rows in green are in the database
    Rows in yellow have been marked as questionable
  • The brackets ([] and {}) in the above examples and templates are not meant to be taken literally
    When you are actually entering data, remove them
  • The Chip ID should be nothing more than the Chip ID/Barcode you entered in the setup
    i.e. do not include "Chip-" at the beginning of the Chip ID
  • The correct header must be in the first three rows for a file to be considered valid
  • Uploading overwrites previous data
  • You will have to reselect the desired file after a failed submission

Changing QC

  • To invalidate a reading, click the checkbox in the "Exclude" column

Flagging for Review

  • Clicking flags the current readout for review
  • Clicking Notes brings up an input dialog where you can explain why this readout was flagged
  • Clicking removes the flag from the current readout

Chip Results

Adding or Editing Chip Test Results

  • Chip Results distill raw data into summaries and important values like half-life
  • First, choose a Readout
    Without a readout, you are unable to specify Assay
  • Summary of Results should be a brief summary of your findings from this chip
    These results can be qualitative
  • Each Result represents a finding from a specific assay
    Clicking Add Result will make another row

Flagging for Review

  • Clicking flags the current results for review
  • Clicking Notes brings up an input dialog where you can explain why these results were flagged
  • Clicking removes the flag from the current results

Models

Organ Models

  • Organ Models illustrate how a microdevice is prepared to mime an organ
  • You can designate the model as being part of the MPS project, EPA project, both, or neither
  • You can associate a number of Protocols with a model
    This allows you to clarify the differences between setups using the same model with different protocols
  • The version name of the protocol can be any 20 characters, but it is recommended that you adhere to a convention such as increasing numbers:
    i.e. v1.0 -> v1.5 -> v2.0
  • Please be sure to upload each protocol as a PDF so that it is more accessible
  • If you need to add a protocol, click the green "Add Protocol" button to add a row in the table
  • If a protocol involves using a different Microdevice, make a new Organ Model using that device
  • You will have to reselect the image file after a failed submission (only when adding a new image)

Microdevices

  • Microdevices describe a device's dimensions along with other characteristics
  • If you are making a plate and intend on using it for plate setups, please fill out the Layout section
  • The Layout section is primarily intended for plates, but can be used for chips as well
  • You will have to reselect image files after a failed submission (only when adding new images)

Compounds

Chemical Data

Viewing Chemical Data

  • The Chemical Data section displays information about pertinent compounds
  • After choosing a compound to view, you can traverse the list in alphanumeric order with the "Next" and "Previous" buttons
  • The labels beneath the image of the compound indicate what project or projects the compound is part of
  • The "Medchem Alerts" field shows whether functional groups that are uncommon in drugs are present in this compound
    If there are Medchem Alerts for a compound, you can click on the flag to see a list of the functional groups

Adding Chemical Data

  • Before entering data yourself, please check ChEMBL (follow the link) for the compound you are interested in
  • If you found one, put the entire ChEMBL ID (including "CHEMBL") into the ChEBML ID field and hit "Retrieve"

Bioactivities

About Bioactivities

  • This Bioactivity data comes from ChEMBL and PubChem
  • Some data from these sources has been marked questionable for various reasons, you can exclude such entries

Using the Quick Search

  • You do not need to fill out every field in this form
    It is only neccessary to fill out one of the three fields: "Compound", "Target", or "Name"
  • Search suggestions will appear for the compounds field
  • This interface can also be accessed on the home page: click the drop down button reading "Global" and switch to "Bioactivities"

Using the Advanced Search

  • Scroll down to see the Advanced Search options
  • Check the checkboxes of desired rows from each section and hit "Generate Table" when you are done
  • Clicking the checkbox at the top of a section will select all currently visible checkboxes in that section
  • To limit the number of choices to those containing specific characters, type in the Filter located above the respective section
  • The Minimum Feature Count Per Record field specifies the number of bioactivities that need to be linked to a given option for it to appear
    e.g. if the min feature count is 10, then the compound "ALDRIN" won't be an option because only 5 bioactivities link to "ALDRIN"
  • Changes to the min feature count are not automatically applied, be sure to hit the "Apply" button if you want to change the min feature count

  • The resulting tables are similar in design to the other tables in the database
  • The "Data Validity" column shows whether or not and why an entry has been marked questionable
  • Please note that not every ChEMBL entry has a corresponding PubChem entry and vice versa

Drug Trials

  • Drug Trials contain manually curated preclinical, clinical, and post-marketing data for compounds
  • After choosing a trial to view, you can traverse the list in alphanumeric order with the "Next" and "Previous" buttons

Adverse Events

Viewing Adverse Events

  • Adverse Events contain automatically acquired data from OpenFDA
  • The "Number of Reports" column indicates the number of reports that list both the compound and adverse event in the row
    Please note that a compound can be listed in a report but not suspected of causing the adverse event
    Likewise, "Number of Reports" in this interface does not contain any information concerning the dose of the drug
  • Only top adverse events for compounds are shown in this interface
  • After choosing a compound to view, you can traverse the list in alphanumeric order with the "Next" and "Previous" buttons
  • While viewing a compound's adverse events, you can create time plots by selecting the "Toggle Plot" button in each row of the table
  • It is also possible to generate tables of adverse event frequencies given a specific time frame by using the interface at the bottom of the page
  • Additional information from OpenFDA such as warnings are sometimes found for a compound, click the "Show" button to view such data
  • OpenFDA is in beta: This data is not intended for clinical use

Compare Adverse Events

  • Make selections by clicking the checkboxes in the furthest left column of the tables
  • Bar graphs will not appear until you have selected at least one adverse event and at least one compound
  • You can save, load, and clear your selections by using the buttons at the top
  • "Reports Among Top" is the number of reports of the most common adverse events for each compound
    Entries with 0 for "Reports Among Top" may still have data from OpenFDA, these entries have a zero value because they are not top adverse events
  • "Estimated Usage" are values derived from "Drug Mentions" listed in the CDC NAMCS and NHAMCS surveys of clinics, out-patient, and ER facilities
  • "Estimated Usage" is used to normalize the number of adverse events reports
    To show the raw counts of adverse event reports, click the "Show Raw Counts" button

Compound Report

  • Compound Report lets you compare compound summaries
  • Make selections by clicking the checkboxes in the furthest left column of the table
  • When you are finished making selections, click "Generate Compound Report"
  • You can save, load, and clear your selections by using the buttons at the top
  • The report shows a summary of compound properties and plots of data for available assays
  • Please note that only normalized data is considered in the plots

Cell Samples

Cell Types

  • Cell Types describe a variety of cells, such as astrocyte or hepatocyte
  • Please be sure to specify the organ, as many cell types differ depending on where they are located
    e.g. hepatic stellate cells differ from pancreatic stellate cells

Cell Origins

  • Cell Origins elaborate on the origin of the cells, for instance whether they are fresh, cryproeserved, or from a cell line
  • Select a Cell Type ONLY if this Origin is SPECIFIC to a certain Cell Type (e.g. a cell line)

Cell Samples

  • Cell Samples describe a specific instance of cells used for experiments
  • If the Cell Origin of the sample is a cell line, please be sure to select the Cell Type first
  • Differentiating cell lots on date received and other criteria can help in quality control if the cells are discovered to be problematic
