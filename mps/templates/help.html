What's New

---

Recent Upgrades to the MPS Database

  • Recent Upgrades to the MPS Database are Described Briefly Below.
  • Integrated Reproducibility Analysis An intra- and inter-study reproducibility analysis tool is now available in the MPS database. The inter-study reproducibility analysis tool can be accessed through the Data Analysis icon. This tool offers the user a wide range of options for grouping and filtering data prior to performing analysis. The intra-study tool can be accessed through the Study Summary Page. For more information on this feature, click here.
  • Matrix Items in a Matrix Matrix Items can now be assigned to a matrix and data can be summarized by matrix. To learn more about matrix and matrix items, review the descriptions in the glossary section of the help. For more information on this feature, click here.
  • Integration of Images and Videos Images and videos can now be associated to a matrix item within a study. Images/videos are provided with metadata explaining what is being displayed. Images/videos are auto-scaled for optimal viewing online, and the unaltered images/videos can be downloaded to perform relative comparisons. Images/videos can be accessed from the main study page (Study Matrices and Items List Page). For more information on this feature, click here.
  • Inter-Study Analysis with Grouping and Filtering The grouping and filtering feature is a powerful tool that offers the user a wide range of options for grouping and filtering data. The tool can be accessed in several places throughout the database, including: through the Data Analysis icon and the Study Summary Page. The tool can be used to customize what data are used (via the filtering feature) and how data will be grouped (via the grouping feature) graphing and analysis. For more information on this feature, click here.
  • Disease Model Portal The disease model portal allows users to quickly determine what information regarding a specific disease is available in the MPS Database, lets the user drill down on data stored in the MPS database (study data and data extracted from the clinicaltrials.gov), and provides users with a links to clinicaltrials.gov where addition drug trial information can be accessed. The disease model portal can be accessed through the Models tab (via the View Diseases option). For more information on this feature, click here.
  • Non-Compound Treatments and/or SettingsThis feature allows the study data provider the ability to enter treatments and/or experimental parameters that are thought to impact the assay data measurements. These, along with the added compounds, can be used to group and filter data to prepare them for visualization and analysis. For more information on this feature, click here.

Searching the Database

Searching Description

  • Conducting a Global Search from the Home Page
    • To perform a Global Search from the Home page:
      • Type text to search in the Global search bar (put a space between terms for multiple term searching)
      • Make preliminary selection (hit enter, click the blue Search button, or mouse down and click the left mouse button)
      • Wait for a list of matching information to appear
      • Click in boxes in Filter section to further limit matching results (optional)
      • Click on link to desired location
  • There are search bars in various locations in the MPS-GUI. Most limit returned results on entry (no hitting enter or a search button is required). To search on multiple terms, include a space between the terms in the search bar.

Overview Information

---

Background

  • The Microphysiology Systems database (MPS-Db) was developed and is maintained by the University of Pittsburgh Drug Discovery Institute (UPDDI).
  • The Microphysiology Systems database (MPS-Db) was designed for the management and analysis of MPS data together with curated reference and clinical data. The MPS-Db can store assay data and associated metadata from a wide variety of plate-based and microfluidic organ models. Data from external reference sources are brought into the MPS to facilitate ease of access to those data and to support integration of data across curated data sources and with study data.

    All study data and study components (e.g., models, devices, cell samples) are manually curated into the MPS Database. Data from some references data sources are both manually and auto-curated. The MPS Database provides tools to analyze and/or visualize data pulled from external data sources, and also provides tools to analyze curated study data with reference data. Viewing study components, study data, and the features of the database query and visualize study and reference data are describing briefing in the tables below.

Permission Structure

  • Role Access to Data and Features
    Registered MPS Users with No Additional Permissions Can view all reference data and associated features. Can view study component lists (e.g., models, devices, cell samples, etcetera). Can view study data in studies that have been designated as open to all registered MPS users, including: viewing matrix item setup, study protocols, images, videos, and study data.
    Data Group Viewers and Users with Additional Permissions Have all the access of registered MPS Users with No Additional Permissions plus access to studies and study data in studies to which they have been granted special access.
    Data Group Editors Have all the access of Data Group Viewers and Users with Additional Permissions plus the ability to edit their studies, including: adding/deleting/editing studies and study details; adding/deleting/editing matrices and matrix items; uploading assay data; and validating matrices, matrix items, and studies.
    Data Group Admins Have all the access of Data Group Editors plus the ability to sign off a study and monitor the acknowledgment status of a study (e.g., if a model developer has reviewed their study data).
    MPS Database Administrators Have all the access of Data Group Admins plus the ability to add/edit/delete new controlled vocabulary to list (e.g., MPS models, Devices, Cell Samples, Locations, excetera). Can change user permissions, including establishing users with acknowledgment access to specific studies.

Uploading Data Points

{#

#}
  • It is possible to upload multiple sets of data points by uploading an Excel file (.xls or .xlsx) or CSV (.csv)
  • To access this interface, go to the desired study's page and click this button: Upload Data
  • Click the following links to download Excel templates:
    Click Here For Upload Template
  • The templates indicate where to place the Chip/Plate IDs
  • The Chip/Plate IDs must be for Chips/Plates in the current study
  • All valid sheets (based on the header) of an Excel file are considered for upload
  • Before uploading data, please be sure to add every assay performed in the section entitled Assays
    You can change this list by going to the Study's edit page and finding the section Assays
    Please see Study

  • The format for data point uploads is as follows:

    Chip ID Cross Reference Assay Plate ID Assay Well ID Day Hour Minute Target / Analyte Subtarget Method / Kit Sample Location Value Value Unit Replicate Caution Flag Exclude Notes
    [Chip ID] [Cross Reference] [Assay Plate ID] [Assay Well ID] [Day] [Hour] [Minute] [Target / Analyte] [Subtarget] [Method / Kit] [Sample Location] [Value] [Value Unit] [Replicate] [Caution Flag] [Leave Blank to Pass] [Notes]
    [Chip ID] [Cross Reference] [Assay Plate ID] [Assay Well ID] [Day] [Hour] [Minute] [Target / Analyte] [Subtarget] [Method / Kit] [Sample Location] [Value] [Value Unit] [Replicate] [Caution Flag] [Any Value in this Column Marks the Reading Invalid] [Notes]
  • File uploads can use either full names ("Albumin") or abbreviations ("ALB") for Target/Analytes
  • Some assays involve transferring media from a device into a plate
    For these assays, specify this information with Assay Plate ID and Assay Well ID
  • Please note that time is split into Day, Hour, and Minute
  • The Sample Location indicates from where in a device the tested sample was acquired
    For instance, if an Organ Model is split into a top and bottom compartment, one might put terms like "Apical" or "Basal" in the Sample Location column
  • The brackets ([] and {}) in the above examples and templates are not meant to be taken literally
    When you are actually entering data, remove them
  • The Chip ID should be nothing more than the Chip ID/Barcode you entered for a Matrix Item
  • The correct header must be in the first three rows for a file to be considered valid
  • Uploading overwrites previous conflicting data (an alert will indicate if data is going to be overwritten)
  • You will have to reselect the desired file after a failed submission

Bioactivities

About Bioactivities (text has not yet been updated)

  • This Bioactivity data comes from ChEMBL and PubChem
  • Some data from these sources has been marked questionable for various reasons, you can exclude such entries

Using the Quick Search

  • You do not need to fill out every field in this form
    It is only neccessary to fill out one of the three fields: "Compound", "Target", or "Name"
  • Search suggestions will appear for the compounds field
  • This interface can also be accessed on the home page: click the drop down button reading "Global" and switch to "Bioactivities"

Using the Advanced Search

  • Scroll down to see the Advanced Search options
  • Check the checkboxes of desired rows from each section and hit "Generate Table" when you are done
  • Clicking the checkbox at the top of a section will select all currently visible checkboxes in that section
  • To limit the number of choices to those containing specific characters, type in the Filter located above the respective section
  • The Minimum Feature Count Per Record field specifies the number of bioactivities that need to be linked to a given option for it to appear
    e.g. if the min feature count is 10, then the compound "ALDRIN" won't be an option because only 5 bioactivities link to "ALDRIN"
  • Changes to the min feature count are not automatically applied, be sure to hit the "Apply" button if you want to change the min feature count
  • The following actions will clear all your selections and cause a refresh:
    Changing between ChEMBL and PubChem, Changing whether questionable are excluded, Changing the min feature count
  • Changing the Target Types or Organisms will clear your Target selections and cause a refresh
  • The data for compounds is simple enough that you can use Filter Compounds without a refresh

Viewing the Table

  • The resulting tables are similar in design to the other tables in the database
  • The "Data Validity" column shows whether or not and why an entry has been marked questionable
  • Please note that not every ChEMBL entry has a corresponding PubChem entry and vice versa

Adverse Events

Viewing Adverse Events (text has not yet been updated)

  • Adverse Events contain automatically acquired data from OpenFDA
  • The "Number of Reports" column indicates the number of reports that list both the compound and adverse event in the row
    Please note that a compound can be listed in a report but not suspected of causing the adverse event
    Likewise, "Number of Reports" in this interface does not contain any information concerning the dose of the drug
  • Only top adverse events for compounds are shown in this interface
  • After choosing a compound to view, you can traverse the list in alphanumeric order with the "Next" and "Previous" buttons
  • While viewing a compound's adverse events, you can create time plots by selecting the "Toggle Plot" button in each row of the table
  • It is also possible to generate tables of adverse event frequencies given a specific time frame by using the interface at the bottom of the page
  • Additional information from OpenFDA such as warnings are sometimes found for a compound, click the "Show" button to view such data
  • OpenFDA is in beta: This data is not intended for clinical use

Compare Adverse Events (text has not yet been updated)

  • Make selections by clicking the checkboxes in the furthest left column of the tables
  • Bar graphs will not appear until you have selected at least one adverse event and at least one compound
  • You can save, load, and clear your selections by using the buttons at the top
  • "Reports Among Top" is the number of reports of the most common adverse events for each compound
    Entries with 0 for "Reports Among Top" may still have data from OpenFDA, these entries have a zero value because they are not top adverse events
  • "Estimated Usage" are values derived from "Drug Mentions" listed in the CDC NAMCS and NHAMCS surveys of clinics, out-patient, and ER facilities
  • "Estimated Usage" is used to normalize the number of adverse events reports
    To show the raw counts of adverse event reports, click the "Show Raw Counts" button
